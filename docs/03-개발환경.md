## 1. 기술 스택

### 서버 (Rust)
| 역할 | 크레이트(후보) | 비고 |
|---|---|---|
| 물리 시뮬 | **rapier2d** (dimforge) | 로봇·공·벽·골 센서, 60Hz step |
| 비동기 런타임 | **tokio** | 이벤트 루프, 타이머 |
| WebSocket/HTTP | **axum** (+ `tokio-tungstenite`) | WS 업그레이드, 정적 서빙(선택) |
| 직렬화 | **serde / serde_json** | JSON 메시지 |
| 로깅 | **tracing** | 개발 관측 |

- 물리는 **서버에서만** 계산(서버 권위). 클라는 렌더만.
- 고정 timestep 60Hz 물리 루프 → 30Hz 스냅샷 브로드캐스트.

### 클라이언트 (브라우저)
| 역할 | 도구 | 비고 |
|---|---|---|
| 언어 | **TypeScript** | |
| 렌더 | **Canvas 2D API** | 사각형 박스 로봇 + 공 + 골대, 라이브러리 불필요 |
| 통신 | **WebSocket API** | 스냅샷 수신 / 명령 송신 |
| 번들러 | **Vite** | 개발 서버(HMR), 빌드 |

- 보간(interpolation) 렌더 루프(requestAnimationFrame).
- 상태 머신(ATTRACT/SELECT/PLAYING/RESULT) 관리.

### (선택·심화) WASM 공유 물리
- Rust 물리 로직을 **wasm-pack** 으로 빌드해 클라에서 재사용(예측/보간 고도화).
- **현재 범위 밖**. 도입 시 "서버·클라 물리를 한 언어로 통일"이라는 어필 포인트.

## 2. 프로젝트 구조 (제안)

```
git.simplectrl/
├─ docs/                    # 설계 문서 (이 폴더)
├─ server/                  # Rust 서버
│  ├─ Cargo.toml
│  └─ src/
│     ├─ main.rs            # 부트스트랩, axum 라우팅, WS 업그레이드
│     ├─ net/               # 세션 관리, 메시지 (de)serialize, 브로드캐스트, net-sim
│     ├─ sim/               # rapier2d 월드, 물리 tick, 충돌/골/아이템 판정
│     ├─ game/              # 경기 상태머신, 3분 타이머, 스코어, 랭킹(인메모리)
│     ├─ control/           # 제어 계층: 전략→행동(FSM)→액추에이터, AI
│     └─ robots/            # 로봇 스탯 정의(8종)
└─ client/                  # 브라우저 클라
   ├─ package.json
   ├─ index.html
   └─ src/
      ├─ main.ts            # 부트스트랩, 상태머신
      ├─ net/               # WebSocket, 메시지 타입, 스냅샷 버퍼, net-sim(클라측)
      ├─ render/            # canvas 렌더, 보간, 이펙트
      ├─ input/             # 키보드/마우스(→ 터치) → 업링크 명령
      └─ ui/                # ATTRACT / SELECT / PLAYING / RESULT 화면
```

- 서버/클라 **메시지 스키마 공유**: JSON 형태를 양쪽에 문서화([02](02-네트워크-프로토콜.md)) — TS 타입과 Rust 구조체를 수기로 대응(또는 스키마 생성 도구는 심화).

## 3. 빌드 / 실행

### 사전 요구
- **Rust** (stable, `rustup`) + `cargo`
- **Node.js** (LTS) + npm/pnpm

### 개발 실행
```bash
# 1) 서버
cd server
cargo run                 # ws://localhost:PORT 대기

# 2) 클라 (별도 터미널)
cd client
npm install
npm run dev               # Vite 개발 서버 (http://localhost:5173)
```
- 브라우저 창(또는 탭) 2개를 열어 각각 BLUE/RED 슬롯 참가 → 1:1 시연.
- 한쪽/양쪽을 AI 모드로 두면 무인 데모(ATTRACT).

### 프로덕션 빌드(선택)
```bash
cd client && npm run build      # 정적 파일 산출
cd server && cargo build --release
```
동작·배포 형태는 [04-동작환경](04-동작환경.md) 참조.

## 4. 개발 편의

- **NET SIM 오버레이**([02](02-네트워크-프로토콜.md) §7): 지연/지터/드랍 토글로 무선 조건 시연.
- **tracing 로그**: tick 시간, 세션 접속/이탈, 브로드캐스트 크기 관측.
- **로봇 스탯 핫스왑**: 스탯을 설정(JSON/상수 테이블)으로 두어 밸런싱 반복 용이.

## 5. 의존성/버전 정책

- Rust/Node는 개발 시점 stable/LTS 사용. `Cargo.lock` / `package-lock` 커밋으로 재현성 확보.
- 외부 서비스·API·DB 의존 없음(데이터는 인메모리) → 환경 세팅이 가볍다.
