# 네트워크 / 프로토콜

> 작성일: 2026-07-03

> 프레이밍: **클라 ↔ 서버 = 로봇 무선 통신 링크** (다운링크=텔레메트리, 업링크=명령)

---

## 1. 링크 모델 (무선화 관점)

```
   [웹 클라이언트]                          [Rust 서버 = 로봇 본체 / 관제]
        │  ── 업링크: 명령(제어 의도) ─────────►  │
        │  ◄──── 다운링크: 텔레메트리(상태) ──────  │
        └─────────── WebSocket 단일 세션 ──────────┘
```

- **다운링크(텔레메트리)** = 30Hz 상태 스냅샷 (로봇/공/디버프/아이템/스코어)
- **업링크(명령)** = 제어 입력(액션/전략/모드전환), **변화 시** 전송
- 실제 로봇의 "관제가 상태를 계속 받고, 조종 명령을 쏜다"와 동일 구조.

---

## 2. 전송 방식

- **WebSocket 단일 채널**, 메시지 인코딩 = **JSON**.
- 근거: 1:1 · 30Hz 규모엔 대역폭·성능 문제 없음. JSON은 디버깅/가시성이 좋아 포트폴리오에 유리.
- WebSocket은 TCP 위 → **무손실·순서보장**(진짜 무선의 손실은 없음). 이를 보완하는 것이 §7 무선 시뮬레이터.
- *(심화, 범위 밖)*: 진짜 비신뢰 전송이 필요하면 **WebTransport(UDP 계열)** 로 교체 가능. 지금은 도입하지 않음.

## 3. 메시지 카탈로그

### 3.1 다운링크 (서버 → 클라)

| 타입 | 시점 | 내용 |
|---|---|---|
| `welcome` | 접속 시 | 클라ID, 현재 경기 상태, 로봇 로스터, 설정(tickRate 등) |
| `state` | 30Hz | 스냅샷 (아래 §4) |
| `event` | 발생 시 | 골 / 경기 시작·종료 / 아이템 획득 / 디버프 적용 (이펙트·사운드용 단발) |
| `ranking` | 경기 종료·갱신 | 로봇별 승률 테이블 |
| `pong` | ping 응답 | 지연 측정용 타임스탬프 |

### 3.2 업링크 (클라 → 서버)

| 타입 | 내용 | 빈도 |
|---|---|---|
| `join` | `{ "slot": "blue"|"red", "robot": "striker-01" }` 슬롯 이어받기 | 단발 |
| `leave` | `{}` 나가기 → AI 인계 | 단발 |
| `input` | `{ "fwd":bool, "back":bool, "turn":-1|0|1, "run":bool, "stop":bool, "rest":bool, "shootCharge":bool, "tackle":bool, "jump":bool }` 액션층 | 변화 시 스트리밍 |
| `strategy` | `{ "mode":"attack"|"defend"|"chase"|"hold" }` 또는 `{ "target":{ "x":number, "y":number } }` | 단발/저빈도 |
| `ctrlmode` | `{ "value":"direct"|"strategy"|"ai" }` 런타임 제어 전환 | 단발 |
| `ping` | `{ "t": number }` 지연 측정 | 1Hz |

모든 메시지는 `t`(타입) 필드로 구분. 순서/지연 판별용 `seq` 또는 타임스탬프를 부가할 수 있다.

## 4. 상태 스냅샷 스키마 (`state`)

```json
{
  "t": "state",
  "phase": "attract" | "playing" | "ended",
  "time": 142,
  "score": { "blue": 1, "red": 0 },
  "robots": [
    {
      "id": "blue",
      "robot": "striker-01",
      "x": 3.1, "y": 1.2, "rot": 0.7,
      "vx": 0.4, "vy": 0.0,
      "ctrl": "human" | "ai",
      "stamina": 0.82,
      "st": ["stun", "slow"],
      "cd": { "tackle": 0.4, "jump": 0.0 }
    },
    { "id": "red",  "robot": "guard-02", "x": 8.4, "y": 2.0, "rot": 3.1, "ctrl": "ai", "stamina": 1.0, "st": [], "cd": {} }
  ],
  "ball": { "x": 5.0, "y": 1.5, "vx": 0.3, "vy": -0.1 },
  "item": { "kind": "boost" | "jam" | null, "x": 5.0, "y": 2.0 }
}
```

- `st` = 활성 상태/디버프 목록: `stun`(이동불가), `shoot_lock`(슛불가), `stop_lock`(정지불가), `tackle_lock`, `jump_lock`, `slow`(감속) 등.
- `cd` = 액션별 쿨다운 잔여(초).
- 좌표는 서버 물리 월드 단위(예: 미터). 클라가 canvas 픽셀로 스케일.

## 5. 처리 방식 (동기화)

```
서버 루프:
  60Hz 물리 tick — 매 tick:
    버퍼된 최신 입력 적용 → rapier2d step → 디버프/충돌/골/아이템 판정
  30Hz 브로드캐스트 — 2 tick마다 전체 스냅샷을 모든 세션에 전송

클라 루프:
  입력 샘플링(프레임마다) → 변화분만 업링크 송신
  스냅샷 2개를 버퍼 → (now - 지연버퍼 ~100ms) 시점으로 보간(interpolation) 렌더
```

- **입력 버퍼링**: 업링크는 비동기 도착 → 서버는 각 슬롯의 "가장 최근 입력"을 다음 tick에 적용.
- **보간만**: 클라는 예측(prediction)/보정(reconciliation) 없이 두 스냅샷 사이를 lerp. 지연/지터가 있어도 부드럽게. (1:1 규모엔 충분, "가볍게" 목표에 부합)
- **서버 권위**: 물리·충돌·디버프·골·아이템·승패 전부 서버 판정. 스턴/잠금 중 업링크가 와도 서버가 무시.

## 6. 연결 수명주기 (무선 = 끊김 대비)

```
접속 → welcome 수신 → 관전(다운링크만)
  join → 슬롯 제어(업링크 추가)
  [끊김 감지] 1초 heartbeat(ping) 무응답 N회 → 클라에 "재연결 중" 표시
  자동 재연결 시도 → 서버는 해당 슬롯을 5초 유예 보관
       재연결 성공 → 슬롯 복귀
       유예 초과   → AI가 슬롯 인계
  leave / 세션 종료 → AI 인계 → ATTRACT 복귀
```

- HUD의 **● connected / tick / ping** = 링크 품질계기판(01 §2.3).
- 슬롯 유예 보관은 어트랙트 모델과 맞물림: 사람이 빠지면 경기가 끊기지 않고 AI가 이어감.

## 7. 무선 현실성 시뮬레이터 (포함)

WebSocket(TCP)은 실제로는 무손실·순서보장이라, "무선 링크가 열악한 조건에서도 견딤"을 보여주기 위한 **개발/시연용 주입기**.

```
┌ NET SIM ────────────────┐
│ 지연   [——●———] 200ms    │
│ 지터   [—●————]  50ms    │
│ 드랍   [●—————]   5%     │
│ [ 켜기 / 끄기 ]           │
└─────────────────────────┘
```

- 메시지를 렌더/처리 전에 인위적으로 **지연 / 지터 / 드랍** 주입(전송 파이프에 삽입).
- 시연 효과:
  - 지연·지터 → **보간 덕에 화면은 부드러움**(보간 가치 증명)
  - 드랍 → 스냅샷이 한둘 빠져도 보간이 메꿈
  - 심하게 → "재연결 중" → 자동 복구(링크 견고성 증명)
- 평소 꺼둠. 시연 시 "지연 200ms 걸어도 이만큼 버팁니다"를 라이브로 시연.
- 구현 위치: 클라측(수신 스냅샷/송신 입력에 딜레이·드랍) 이 가장 간단. 서버측 per-connection 주입도 가능.
- 참고: 폰이 실제 와이파이/셀룰러로 붙으면 **진짜 지연·지터**가 생겨 시뮬레이터 없이도 자연스러운 무선 시연이 됨. 시뮬레이터는 극한 조건을 임의로 거는 용도.

---

## 8. 요약 (확정 사항)

- 전송: WebSocket 단일 세션, JSON
- 다운링크: 30Hz `state` 스냅샷 + 단발 `event`/`ranking`/`welcome`/`pong`
- 업링크: `join`/`leave`/`input`/`strategy`/`ctrlmode`/`ping`
- 서버 60Hz 물리 tick, 30Hz 브로드캐스트, 클라 보간(예측/보정 없음)
- 재연결 5초 유예 → AI 인계, heartbeat로 링크 품질 표시
- 무선 시뮬레이터 포함(개발/시연용)
